# 問題パターン識別ガイド

## 概要

このガイドでは、LeetCodeやAtCoderの問題を効率的に解くための**問題パターンの識別方法**を解説します。問題を見たときに、どのアルゴリズムやデータ構造を使うべきかを素早く判断できるようになることが目標です。

## 問題パターンの分類

### 1. 配列・文字列問題

#### 識別ポイント
- 配列や文字列の操作が必要
- 検索、ソート、マッチングが主な操作

#### 典型的な問題
- 要素の検索、カウント
- 部分文字列・部分配列の処理
- アナグラム、パリンドローム

#### 推奨アルゴリズム
- **ハッシュテーブル**: 要素の存在確認、カウント
- **二ポインタ**: ソート済み配列の処理
- **スライディングウィンドウ**: 連続する部分配列・部分文字列
- **ソート**: 順序の整理

#### 例
- Two Sum → ハッシュテーブル
- Longest Substring Without Repeating Characters → スライディングウィンドウ
- Group Anagrams → ハッシュテーブル + ソート

### 2. グラフ問題

#### 識別ポイント
- ノードとエッジの関係が重要
- 探索、最短経路、サイクル検出が必要

#### 典型的な問題
- 連結成分の検出
- 最短経路の計算
- サイクルの検出
- トポロジカルソート

#### 推奨アルゴリズム
- **DFS/BFS**: グラフの探索
- **Dijkstra**: 非負重みの最短経路
- **Bellman-Ford**: 負重みの最短経路
- **Union-Find**: 連結成分の管理
- **トポロジカルソート**: 依存関係の解決

#### 例
- Number of Islands → DFS/BFS
- Course Schedule → トポロジカルソート
- Network Delay Time → Dijkstra

### 3. 木問題

#### 識別ポイント
- 階層構造を持つデータ
- 探索、パス、構築が必要

#### 典型的な問題
- 木の探索（DFS/BFS）
- パスの検索
- 木の構築
- バランスチェック

#### 推奨アルゴリズム
- **DFS**: 深さ優先探索
- **BFS**: 幅優先探索
- **再帰**: 木の操作
- **スタック**: 反復的DFS

#### 例
- Maximum Depth of Binary Tree → DFS
- Binary Tree Level Order Traversal → BFS
- Validate Binary Search Tree → DFS + 範囲チェック

### 4. 動的計画法問題

#### 識別ポイント
- 最適化問題
- 重複する部分問題
- 選択問題（選ぶ/選ばない）

#### 典型的な問題
- 最適な選択を求める
- 組み合わせの数
- 最長/最短のパス
- コストの最小化/最大化

#### 推奨アルゴリズム
- **動的計画法**: メモ化、ボトムアップ
- **バックトラッキング**: 全ての組み合わせを生成

#### 例
- Climbing Stairs → DP（Fibonacci）
- House Robber → DP
- Coin Change → DP
- Longest Increasing Subsequence → DP

### 5. 貪欲法問題

#### 識別ポイント
- 局所最適選択が全体最適解
- 各ステップで最適な選択を行う

#### 典型的な問題
- スケジューリング問題
- 選択問題
- 区間問題

#### 推奨アルゴリズム
- **貪欲法**: 各ステップで最適な選択

#### 例
- Best Time to Buy and Sell Stock → 貪欲法
- Maximum Subarray → 貪欲法（Kadane）
- Non-overlapping Intervals → 貪欲法

### 6. バックトラッキング問題

#### 識別ポイント
- 全ての可能な解を探索
- 制約を満たす解を探す
- 組み合わせ・順列の生成

#### 典型的な問題
- N-Queens
- Sudoku Solver
- Generate Parentheses
- Combination Sum

#### 推奨アルゴリズム
- **バックトラッキング**: 再帰的探索

#### 例
- Generate Parentheses → バックトラッキング
- N-Queens → バックトラッキング
- Sudoku Solver → バックトラッキング

### 7. スライディングウィンドウ問題

#### 識別ポイント
- 連続する部分配列・部分文字列
- 固定サイズまたは可変サイズのウィンドウ

#### 典型的な問題
- 固定サイズの部分配列
- 条件を満たす最長/最短の部分配列
- 文字列の部分文字列

#### 推奨アルゴリズム
- **スライディングウィンドウ**: 二ポインタ

#### 例
- Maximum Sum Subarray of Size K → 固定サイズウィンドウ
- Longest Substring Without Repeating Characters → 可変サイズウィンドウ
- Minimum Window Substring → 可変サイズウィンドウ

### 8. 二分探索問題

#### 識別ポイント
- ソート済み配列
- 検索、境界の検出
- 最適値の探索

#### 典型的な問題
- ソート済み配列での検索
- 境界の検出
- 最適値の探索

#### 推奨アルゴリズム
- **二分探索**: O(log n)検索

#### 例
- Binary Search → 基本的な二分探索
- Search in Rotated Sorted Array → 回転配列の二分探索
- Find Peak Element → 二分探索の応用

### 9. スタック・キュー問題

#### 識別ポイント
- LIFO/FIFOの操作
- 括弧の対応
- モノトニックスタック

#### 典型的な問題
- 括弧の対応チェック
- 次の大きい要素
- ヒストグラムの最大矩形

#### 推奨アルゴリズム
- **スタック**: LIFO操作
- **キュー**: FIFO操作
- **モノトニックスタック**: 単調性を保持

#### 例
- Valid Parentheses → スタック
- Daily Temperatures → モノトニックスタック
- Largest Rectangle in Histogram → モノトニックスタック

### 10. ヒープ問題

#### 識別ポイント
- 優先度に基づいた処理
- トップK問題
- マージ問題

#### 典型的な問題
- K番目に大きい/小さい要素
- マージK個のソート済みリスト
- 頻度の高い要素

#### 推奨アルゴリズム
- **ヒープ**: 優先度付きキュー

#### 例
- Kth Largest Element → ヒープ
- Merge K Sorted Lists → ヒープ
- Top K Frequent Elements → ヒープ

## 問題の読み方

### 1. 制約の確認
- **配列のサイズ**: データ構造の選択に影響
- **値の範囲**: 座標圧縮が必要か判断
- **時間制限**: アルゴリズムの選択に影響

### 2. 操作の確認
- **クエリの種類**: 範囲クエリ、点更新など
- **操作の頻度**: 各操作の時間計算量を考慮
- **操作の順序**: 時系列順か、任意の順序か

### 3. パターンの識別
- **類似問題**: 過去に解いた問題と比較
- **データ構造**: 適切なデータ構造を選択
- **アルゴリズム**: 適切なアルゴリズムを選択

## 実践的なアプローチ

### ステップ1: 問題の理解
1. 問題文を読む
2. 制約を確認
3. 例を理解

### ステップ2: パターンの識別
1. データ構造を特定（配列、グラフ、木など）
2. 操作を特定（検索、ソート、探索など）
3. パターンを特定（スライディングウィンドウ、DPなど）

### ステップ3: アルゴリズムの選択
1. パターンに基づいてアルゴリズムを選択
2. 時間計算量を確認
3. 実装の複雑さを考慮

### ステップ4: 実装
1. アルゴリズムを実装
2. エッジケースを処理
3. テスト

## よくある間違い

### 1. パターンの誤認識
- **問題**: 間違ったパターンを識別
- **解決策**: 問題文と制約を慎重に読む
- **注意**: 類似問題と混同しない

### 2. 過剰な複雑化
- **問題**: 複雑なアルゴリズムを選択
- **解決策**: シンプルなアプローチから始める
- **注意**: 必要以上に複雑にしない

### 3. データ構造の誤選択
- **問題**: 適切でないデータ構造を選択
- **解決策**: 各操作の時間計算量を考慮
- **注意**: 操作の頻度を考慮する
