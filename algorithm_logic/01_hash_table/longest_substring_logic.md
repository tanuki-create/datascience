# Longest Substring - ハッシュテーブルでの応用

## 問題概要

この問題は、ハッシュテーブルとスライディングウィンドウを組み合わせた典型例です。詳細な解説は[スライディングウィンドウ](../03_sliding_window/longest_substring_no_repeat_logic.md)を参照してください。

## ハッシュテーブルとの関連

Longest Substring Without Repeating Charactersは、ハッシュテーブル（ハッシュセット）を**補助データ構造**として使用する良い例です。

### ハッシュテーブルの役割

1. **重複の検出**: ウィンドウ内の文字の存在をO(1)で確認
2. **状態管理**: ウィンドウ内にどの文字が存在するかを効率的に管理
3. **高速な削除**: ウィンドウを縮小する際に、文字をO(1)で削除

### 他のアプローチとの比較

**配列を使った方法（非効率）**:
- 文字の存在確認にO(n)の時間がかかる
- ウィンドウ内の文字を線形探索する必要がある

**ハッシュセットを使った方法（効率的）**:
- 文字の存在確認がO(1)
- ウィンドウの拡張・縮小が効率的

## まとめ

この問題は、ハッシュテーブルが**補助データ構造**として強力であることを示しています。スライディングウィンドウと組み合わせることで、O(n)の時間計算量で最長の部分文字列を見つけることができます。
