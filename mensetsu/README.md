## 使い方

各ドキュメントは、面接で説明できる具体的な実践例を含んでいます。以下のポイントを意識して読み進めてください：

1. **具体的なコード例**: 実際のコード例を通じて理解を深める
2. **Before/After**: 改善前後の比較で効果を理解する
3. **実践例**: 面接で説明できる具体的な経験談の例

## 面接での説明のポイント

### STAR法を活用

- **Situation（状況）**: どのような状況・課題があったか
- **Task（課題）**: 何を解決する必要があったか
- **Action（行動）**: どのような技術・手法を使ったか
- **Result（結果）**: どのような成果が得られたか

### 例

「以前のプロジェクトで、ECサイトの注文履歴画面でN+1問題によりページ読み込みに5秒かかっていました（Situation）。ユーザー体験を改善する必要がありました（Task）。JOINを使用して1回のクエリに集約し、適切なインデックスを追加しました（Action）。その結果、ページ読み込み時間を0.3秒に短縮し、ユーザー満足度が向上しました（Result）。」
